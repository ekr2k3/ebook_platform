# 4. Thiết kế DB

Trong Sprint 1, chưa cần thiết kế toàn bộ hệ thống. Chỉ cần các bảng phục vụ trực tiếp cho upload, preview, library và quản lý người dùng.

## 4.1. Bảng `users`

Lưu thông tin người dùng.

| Trường     | Kiểu dữ liệu | Ghi chú                |
| ---------- | ------------ | ---------------------- |
| id         | BIGINT       | PK                     |
| username   | VARCHAR      | duy nhất               |
| email      | VARCHAR      | duy nhất               |
| password   | VARCHAR      | mã hóa                 |
| role       | VARCHAR      | BUYER / SELLER / ADMIN |
| status     | VARCHAR      | ACTIVE / INACTIVE      |
| created_at | DATETIME     | thời gian tạo          |

---

## 4.2. Bảng `ebooks`

Lưu thông tin cơ bản của ebook.

| Trường      | Kiểu dữ liệu | Ghi chú                             |
| ----------- | ------------ | ----------------------------------- |
| id          | BIGINT       | PK                                  |
| title       | VARCHAR      | tên sách                            |
| description | TEXT         | mô tả                               |
| author      | VARCHAR      | tác giả                             |
| format      | VARCHAR      | PDF / EPUB                          |
| seller_id   | BIGINT       | FK tới users                        |
| file_url    | VARCHAR      | đường dẫn file trên Google Drive    |
| cover_url   | VARCHAR      | ảnh bìa nếu có                      |
| file_size   | BIGINT       | dung lượng file                     |
| previewable | BOOLEAN      | có cho preview hay không            |
| status      | VARCHAR      | DRAFT / VALID / INVALID / PUBLISHED |
| created_at  | DATETIME     | thời gian tạo                       |
| updated_at  | DATETIME     | thời gian cập nhật                  |

---

## 4.3. Bảng `ebook_validation_logs`

Lưu kết quả kiểm tra file.

| Trường            | Kiểu dữ liệu | Ghi chú            |
| ----------------- | ------------ | ------------------ |
| id                | BIGINT       | PK                 |
| ebook_id          | BIGINT       | FK tới ebooks      |
| validation_status | VARCHAR      | VALID / INVALID    |
| error_message     | TEXT         | thông báo lỗi      |
| checked_at        | DATETIME     | thời gian kiểm tra |
- Dùng để lưu lại log khi người dùng upload file (bảng này thực ra cũng không cần thiết lắm, nó chỉ có mục đích theo dõi, không tham gia vận hành)

## 4.4. Bảng `user_ebooks`

Lưu danh sách sách mà người dùng được sở hữu hoặc có quyền truy cập.

| Trường      | Kiểu dữ liệu | Ghi chú            |
| ----------- | ------------ | ------------------ |
| id          | BIGINT       | PK                 |
| user_id     | BIGINT       | FK tới users       |
| ebook_id    | BIGINT       | FK tới ebooks      |

Trong Sprint 1, bảng này dùng để phục vụ **library cơ bản**. Giai đoạn này có thể giả lập dữ liệu mà chưa cần hoàn chỉnh phần payment.

# 6. Danh sách endpoint

## 6.1. Nguyên tắc phân loại

- **Public**: người chưa đăng nhập vẫn truy cập được
    
- **Private**: yêu cầu đăng nhập
    
- **Role-based private**: yêu cầu đúng vai trò
    

---

## 6.2. Endpoint cho upload ebook

### `GET /seller/ebooks/upload`

- Loại: Private
    
- Quyền: SELLER
    
- Mục đích: hiển thị form upload ebook
    
- Method: GET
    

### `POST /seller/ebooks/upload`

- Loại: Private
    
- Quyền: SELLER
    
- Mục đích: nhận dữ liệu upload ebook
    
- Method: POST
    
- Dữ liệu nhận:
    
    - title
        
    - description
        
    - author
        
    - file
        
- Kết quả:
    
    - thành công: chuyển sang trang chi tiết hoặc preview
        
    - thất bại: quay lại form và báo lỗi
        

---

## 6.3. Endpoint cho xem chi tiết ebook

### `GET /ebooks`

- Loại: Public
    
- Quyền: mọi người
    
- Mục đích: hiển thị danh sách ebook
    
- Method: GET
    

### `GET /ebooks/{id}`

- Loại: Public
    
- Quyền: mọi người
    
- Mục đích: hiển thị chi tiết ebook
    
- Method: GET
    

---

## 6.4. Endpoint cho preview

### `GET /ebooks/{id}/preview`

- Loại: Public hoặc Private nhẹ
    
- Quyền:
    
    - buyer: xem preview giới hạn
        
    - seller sở hữu sách: có thể xem bản preview đầy đủ hơn
        
- Mục đích: xem thử ebook
    
- Method: GET
    

### `GET /seller/ebooks/{id}/preview`

- Loại: Private
    
- Quyền: SELLER, chỉ với sách do seller đó upload
    
- Mục đích: seller xem preview trước khi publish
    
- Method: GET
    

---

## 6.5. Endpoint cho library

### `GET /library`

- Loại: Private
    
- Quyền: BUYER
    
- Mục đích: hiển thị thư viện sách đã sở hữu
    
- Method: GET
    

---

## 6.6. Endpoint hỗ trợ file

### `GET /files/ebooks/{id}/view`

- Loại: Private hoặc có kiểm soát
    
- Quyền: tùy luồng
    
- Mục đích: lấy nội dung file để hiển thị preview/viewer
    
- Method: GET
    
- Ghi chú:
    
    - không trả đường link download trực tiếp
        
    - nên đi qua backend để kiểm soát quyền

# 8. Thiết kế nghiệp vụ chi tiết

## 8.1. Nghiệp vụ upload ebook

### File chấp nhận

- Chỉ chấp nhận:
    
    - `.pdf`
        
    - `.epub`
        

### Dung lượng tối đa

- Đề xuất Sprint 1:
    
    - tối đa **50 MB**
        
- Lý do:
    
    - đủ cho demo
        
    - giảm rủi ro upload chậm / lỗi lớn
        

### Luồng xử lý

1. Seller truy cập form upload
    
2. Nhập thông tin ebook
    
3. Chọn file
    
4. Hệ thống kiểm tra:
    
    - có file hay không
        
    - định dạng đúng không
        
    - dung lượng có vượt giới hạn không
        
5. Nếu hợp lệ:
    
    - upload file lên Google Drive
        
    - lưu metadata vào MySQL
        
    - tạo log validation
        
6. Nếu không hợp lệ:
    
    - trả lỗi ra giao diện
        

### Trường hợp lỗi

- Không chọn file
    
    - báo: “Vui lòng chọn file ebook”
        
- Sai định dạng
    
    - báo: “Chỉ hỗ trợ file PDF hoặc EPUB”
        
- Quá dung lượng
    
    - báo: “Dung lượng file vượt quá giới hạn cho phép”
        
- Upload Google Drive thất bại
    
    - báo: “Không thể lưu file, vui lòng thử lại sau”
        
- Lưu DB thất bại sau khi upload file thành công
    
    - tạm thời ghi log lỗi và thông báo thất bại
        
    - có thể phải xóa file đã upload để tránh rác dữ liệu
        

---

## 8.2. Nghiệp vụ validate file

### Mức validate trong Sprint 1

- Validate đơn giản, đồng bộ
    
- Chưa xử lý sâu như parse toàn bộ cấu trúc EPUB phức tạp
    

### Nội dung kiểm tra

- Kiểm tra đuôi file
    
- Kiểm tra content type cơ bản
    
- Thử mở file để xác nhận file không hỏng
    
- Với PDF:
    
    - thử load file bằng viewer
        
- Với EPUB:
    
    - kiểm tra file có thể đọc được ở mức cơ bản
        

### Kết quả

- VALID
    
- INVALID + thông báo lỗi
    

---

## 8.3. Nghiệp vụ preview

### Buyer được xem bao nhiêu

- Buyer chưa mua được xem **5–10% nội dung**
    
- Trong Sprint 1 có thể áp dụng đơn giản:
    
    - PDF: hiển thị vài trang đầu
        
    - EPUB: hiển thị vài chương/đoạn đầu
        

### Seller có được xem full preview không

- Seller là người upload sách được xem preview rộng hơn để kiểm tra nội dung
    
- Tuy nhiên ở Sprint 1 có thể chỉ cần:
    
    - seller xem nhiều hơn buyer
        
    - chưa nhất thiết full 100% nếu kỹ thuật chưa sẵn sàng
        

### Không cho download trực tiếp bằng cách nào

- Không hiển thị link file gốc
    
- Không public URL Google Drive ra ngoài
    
- Nội dung preview đi qua backend/controller
    
- Chỉ render trên iframe/viewer
    
- Ẩn nút download nếu dùng viewer
    

Lưu ý:

- Đây chỉ là mức bảo vệ cơ bản
    
- Chưa phải DRM mạnh

## 8.4. Nghiệp vụ library

### Điều kiện để sách xuất hiện trong thư viện

- User phải đăng nhập
    
- User có bản ghi trong `user_ebooks`
    

### Sprint 1 xử lý đơn giản

- Có thể seed sẵn dữ liệu `user_ebooks`
    
- Hoặc giả lập việc user đã sở hữu sách
    
- Chưa cần tích hợp payment ở sprint này
    

### Hiển thị

- Danh sách sách đã sở hữu
    
- Có thể bấm vào xem chi tiết hoặc preview/đọc
